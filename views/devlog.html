2026-05-15 — Arquitectura 50/50: Mapas JSON + 2D top-down

Corrección de la arquitectura del motor: el mundo no es 100% raycaster, sino 50% 2D top-down + 50% Raycasting pseudo-3D.

Cambios estructurales:

  • Mapas como JSON (public/maps/inicio.json, cueva.json): cada mapa contiene nombre, modo (2d o ray), tiles, colores, salidas y datos del jugador.
  • Transición visual con fade out/in (1s total) al cambiar de mapa por puertas o eventos. Overlay negro con curva easeInOutQuad.
  • Renderización 2D top-down: cámara centrada en el jugador con scroll suave, tiles de 32×32 coloreados según tileColors, puertas con brillo pulsante, y jugador como cuadrado azul con indicador de dirección.
  • Jugador en 2D: movimiento cardinal (WASD en 4/8 direcciones), sin rotación, con animación de bob (±2px senoidal).
  • Cámara (camera.js): viewport que sigue al jugador y se clampa a los bordes del mapa.
  • Nuevo módulo camera.js para la gestión de viewport 2D.
  • COLORES.md en public/maps/ documentando los placeholders de tiles.
  • El raycaster existente se mantiene intacto para los mapas en modo ray.

2026-05-15 — Corrección de bugs y estabilización del motor

Ronda de fixes tras detectar errores en producción durante el testeo del modo 2D y la transición entre mapas:

  • Camera plane dinámico — planeX/planeY se calculan rotando el vector dirección del jugador en vez de estar hardcodeados a (0, 0.66). Esto permite que mapas ray con dirección de spawn distinta tengan el FOV correcto.
  • Race condition en transición — El await loadMap() dentro del game loop suspendía el ciclo de requestAnimationFrame, congelando la animación de fade. Reemplazado por .then() para que el RAF cycle nunca se interrumpa y la transición se reproduzca fluidamente.
  • Facing diagonal corregido — En modo 2D, al presionar dos teclas simultáneamente (ej: W+A) la dirección facing prioriza un eje cardinal (horizontal primero) en vez de apuntar en diagonal.
  • Brillo de puerta dinámico — El color del brillo pulsante de las puertas ahora se lee del tile real en la posición de salida (map.tiles[exit.tileY][exit.tileX]), no del ID hardcodeado 2. Funciona con cualquier tile que actúe como puerta.
  • Rutas absolutas en exits — Los target de los JSON ahora usan rutas absolutas (/maps/cueva.json) para que el fetch funcione independientemente de la URL base de la página.
  • Error handling en Map.load — Validación HTTP (if (!res.ok)) que lanza un error descriptivo en vez de intentar parsear HTML de error 404 como JSON.
  • Guard null en renderer — render() retorna temprano si Map.current === null para evitar un crash en cascada (Cannot read properties of null) tras un error de carga de mapa.

2026-05-15 — Renumeración de tiles: 0 = vacío, capas en mapas

Problema: El tile ID 0 se usaba tanto para "pasto" (tile válido con sprite) como para "vacío" (sin tile), causando que la capa terreno no renderizara nada porque drawLayer salta los tiles con valor 0.

Solución: Renumeración completa de IDs en inicio.json y cueva.json:

  • Tile 0 = vacío (siempre transitable, nunca se renderiza)
  • Tile 1 = pasto (era 0)
  • Tile 2 = pared_piedra (era 1)
  • Tile 3 = puerta (era 2)
  • Tile 4 = agua (era 3)

Fix en map.js: isSolid() ahora retorna false para tile 0 (vacío), ya que al quitar tileSprites["0"] el default return true trataría el espacio vacío como sólido. No se tocó el render.

2026-05-16 — Colisión circular, piso texturizado, fix de paredes y separación de capas

Colisión circular: El jugador y NPCs ahora usan detección de colisión circular con radio 0.5 (medio tile = entidad de 1 tile de diámetro). Nueva función Map.checkCircleCollision(cx, cy, radius) que verifica overlap círculo vs rectángulo de cada tile. El jugador no puede atravesar NPCs/enemigos.

  • map.js — checkCircleCollision(), isWall() (solo estructura), isSolid() ahora chequea estructura + terreno, checkExits() por distancia circular, _tileInBounds(), migrateMapData() divide mundo en estructura/terreno.
  • player.js — COLLISION_RADIUS: 0.5, _circleBlocked() (tiles + entidades), move() usa colisión circular con wall sliding.

Piso texturizado: Floor-casting con texturas reales del atlas en modo ray. Por cada píxel abajo del horizonte se calcula la coordenada mundial, se lee el tile de terreno y se dibuja el texel correspondiente del spritesheet via ImageData. Los tiles decorativos no-sólidos de estructura se blendan como overlay.

Fix de paredes transparentes: Las paredes Y-side (norte-sur) usaban ctx.globalAlpha = 0.6 para sombrear, lo que hacía la textura semi-transparente (se veía el fondo a través). Reemplazado por overlay rgba(0,0,0,0.4) — las paredes ahora se ven sólidas en todas las direcciones.

Separación de capas: La capa legacy mundo ahora se divide automáticamente al cargar el mapa: tiles de pared van a estructura (solo estos detienen el raycaster), agua y decoraciones van a terreno. El agua bloquea al jugador (vía isSolid()) pero no se renderiza como pared.

Documentación: Actualizados README.md, gamecore/README.md, COLORES.md, tareas.json y hitos.json reflejando los nuevos sistemas.

2026-05-16 — Bug fixes post-colisión circular

Corrección de 3 bugs detectados tras la implementación de colisión circular y piso texturizado:

  • Fix horizonte de 1px negro (renderer.js): El floor-casting arrancaba en halfH pero p <= 0 skipeaba la fila del horizonte, dejando un píxel sin dibujar. Cambiado a startY = max(halfH + 1, wallBottom) para que el piso empiece limpiamente abajo del horizonte.
  • Fix posible loop infinito en DDA (raycaster.js): El while loop del raycaster no tenía límite de iteraciones. Si un mapa no tenía bordes de pared, el rayo podía avanzar indefinidamente. Agregado maxSteps = mapWidth * 2 como tope.
  • Fix null guard en _circleBlocked (player.js): Si _circleBlocked() se llamaba sin mapa cargado, acceder a Map.current.characters lanzaba TypeError. Agregado if (!Map.current) return true al inicio.

2026-05-17 — Multi-atlas, sistema de direcciones y Fase 2 de herramientas

Multi-atlas: Migración de un único atlas.png a múltiples atlas por categoría (atlas_mundo.png, atlas_entidades.png, atlas_ui.png). build-atlas.js agrupa entidades por campo atlas y genera un PNG+JSON por grupo. sprite.js implementa búsqueda cross-atlas y getters backward-compatibles (Sprite.atlas retorna el atlas mundo).

Direcciones en sprites: Soporte para directions: "4dir" | "8dir" con layout en el spritesheet (3 o 5 bloques de frames). Mirror automático para left (flip horizontal del bloque right). getFrame(entityId, frameIndex, direction) y dirFrames en atlas JSON.

TileW/TileH y halfBlock: Entidades ahora declaran tileW y tileH (tamaño en tiles). build-atlas.js genera frameW = tileW * tileSize, frameH = tileH * tileSize. halfBlock renderiza a mitad de altura visual (shift down en raycaster, medio frame en 2D). raycaster.js _addBillboard usa tileW/tileH para dimensiones de billboard. renderer.js usa frameW/frameH en draw2D y drawLayer.

Adaptación de las 5 DevTools a multi-atlas:

  • visor-atlas — Carga todos los grupos en canvas apilados con encabezados, selector de atlas, etiquetas atlas por entidad. Detail panel muestra dirFrames, tileW/tileH, halfBlock. Preview con selector de dirección y mirror.
  • inspector-mapa — loadAtlases() reemplaza loadExternalPalette(). Imágenes por grupo para tiles/exits/paleta. Etiquetas atlas en dropdown de entidad. Selector de dirección en modal de personajes/enemigos con flecha direccional en canvas.
  • creador-tiles — Selector de atlas en propiedades. Campos tileW/tileH/directions/mirror/halfBlock/blockVision/halfSolid guardados en entity.js. spriteToBase64() genera el canvas con el tamaño correcto según direcciones. Restaura todos los campos al editar.
  • cortador-texturas — Campos tileW/tileH/directions/mirror/halfBlock en el formulario de selección. Guardados en entityData al persistir. Restaurados al seleccionar.
  • editor-hud — loadAtlasInfo() carga todos los grupos. drawImage() usa atlasImages[s.atlasName]. Etiquetas atlas en dropdown de sprite.

Renderización halfBlock en 2D: drawLayer y draw2D (player, characters, enemies) ahora reducen la altura dibujada a la mitad cuando info.halfBlock es true, posicionando la parte visible sobre el tile.

Entidades actualizadas: player configurado con tileH:2, directions:"4dir", mirror:true. Todas las entidades existentes tienen campo atlas asignado (mundo o entidades).
